# 关卡 06：Memory（短期 + 长期）

## 目标

- 记忆服务化，并控制污染风险。

## 推荐执行方式（先快后深）

## Level 1（30-60 分钟）：AI 生成最小记忆模块

### 直接给 AI 的提示词（可复制）

```text
帮我生成一个最小 memory 模块：
1) 短期记忆摘要（最近 N 轮）
2) 长期记忆存储与召回（按用户）
3) 写入前过滤敏感信息和注入文本
4) 提供 3 个测试（写入、召回、污染阻断）
```

### 你只做 3 件事

- 跑通多轮对话写入与召回。
- 验证不同用户隔离。
- 检查污染样例是否被阻断。

## Level 2（1-2 小时）：工程加固任务

- 增加写入策略（何时写、写哪些字段、置信度）。
- 增加召回策略（按主题、时间、权重）。
- 增加记忆 TTL 或版本管理。
- 增加 10 条多轮回归样例。

## Level 3（2-4 小时）：面试可讲深度

- 增加记忆质量评估（命中率、一致性、污染率）。
- 增加人工纠偏机制（删除或修正错误记忆）。
- 增加审计轨迹（谁写入、为何写入、写入来源）。
- 产出 `memory_policy.md`。

## 产物

- `src/memory/store.py`
- `src/memory/policy.py`
- `docs/memory_policy.md`

## 验收

- L1：同用户偏好可复用。
- L2：多轮样例表现稳定。
- L3：可解释记忆写入与污染防护策略。
