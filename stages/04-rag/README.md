# 关卡 04：RAG（含注入防护）

## 目标

- 让回答具备来源引用与外部知识接入能力。

## 推荐执行方式（先快后深）

## Level 1（30-60 分钟）：AI 生成最小 RAG

### 直接给 AI 的提示词（可复制）

```text
帮我生成最小 RAG 模板：
1) 文档切分 + 向量检索（top-k）
2) 回答时返回 sources[]（标题、路径、片段）
3) 增加一条 prompt injection 过滤规则
4) 提供 3 个测试（命中、未命中、注入文本）
```

### 你只做 3 件事

- 准备小规模知识库文档。
- 跑通检索问答。
- 核对 sources 字段和过滤结果。

## Level 2（1-2 小时）：工程加固任务

- 增加检索重排（rerank）或多路召回。
- 增加 chunk 策略对比（长度/重叠参数）。
- 增加注入规则白名单与拒绝响应模板。
- 增加 20 条问答回归样例。

## Level 3（2-4 小时）：面试可讲深度

- 增加检索评估指标（命中率、MRR、引用正确率）。
- 增加数据快照版本（可回滚知识库）。
- 增加对抗样例集（指令污染、伪造来源、敏感词）。
- 产出 `rag_eval_report.md`。

## 产物

- `src/rag/retriever.py`
- `src/rag/injection_filter.py`
- `evals/rag_cases.json`

## 验收

- L1：回答可附来源且可追溯。
- L2：20 条样例中注入文本不被执行。
- L3：有检索指标与对抗测试报告。
